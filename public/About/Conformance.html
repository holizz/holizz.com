Document Conformance

Valid Everything 1.0!

  • Valid XHTML 1.1 served as application/xhtml+xml.
  • Valid XHTML 1.0 (following most of the HTML Compatibility Guidelines) served as text/html.
  • Valid Atom served as application/atom+xml.
  • Valid CSS served as text/css.

Why?

Why is it all valid?

Because I have nothing better to do, I like to implement things for the sake of doing so, and I want to live in a world where markup is valid and not tag soup.

Why XHTML 1.0 instead of HTML 4.01?

Lazyness. XHTML 1.0 is one line of sed and one line of ed away from XHTML 1.1.

All the content on my site is XML. From the CMS with tables filled with XML external parsed entities to the XHTML or Atom the user sees. All XML. So I decided I’d rather use the HTML-compatible XHTML 1.0 than use (gah) SGML.

Why Atom instead of RSS?

[T]here are 7 different and incompatible versions of RSS .

So much choice! Which do I choose? I don’t know… Thus I chose none.

Another factor was that I tried RSS 1.0 (since I was rather into RDF at the time) and due to feed parsers using XML parsers when they should have been using RDF parsers (since RSS 1.0 is RDF) it worked very badly in most RSS readers and validators (and I wasn’t planning on changing my valid feed to make it work with shitty XML-based RSS parsers). And so I decided Atom was the best way.
