holizz.com

Right. Last URI change.

If you are reading this from holizz.com you may already realise that I have a new domain. And if you’re not, then go to holizz.com and see how ultimately cool it is.

I also have a new URI for me, as well as for my blog. The URI for me is http://holizz.com/Holizz/, which allows one to write nice triplets about me, such as:

<http://holizz.com/Holizz/> <#is> <#gay>.

Okay, so maybe I cheated a little. I’ll look up ‘is’ and ‘gay’ in Dublin Core one day.

And with my new URIs comes a new CMS (not to forget the new TLA). It’s based on the original Sunc databases but this one’s in Ruby and it generates a static archive.

It generates everything with REXML therefore if my markup is bad XHTML then it excepts and I have to correct it, which is good. Since it generates a static archive rather than serves a dynamic one, it can take longer in its algorithms. Thus it won’t strain the server if I want to implement a complex algorithm. Also, it’s been made in such a way that it should be reasonably effortless to make it output in RSS or RDF later on.

So far it only generates the individual pages and the main index page; no yearly, monthly, or daily indeces.

But yes, new domain, new hosting, new URIs. How better could life get? Lectures tomorrow start at 1pm. Woop.
