Now with Atom 0.3

After getting my RSS 1.0 feed working it was trivial to make my Atom feed validate.

The XSLT parser kept putting the XHTML namespace in the root element which the validator didn’t like. It was a simple matter of skimming the specification until I found what I wanted: xsl:exclude-result-prefixes="xhtml" in the xsl:stylesheet element.

So as with the RSS 1.0, the Atom feed is available here and if your user agent doesn’t request a content-type, you can get it here.

I was having trouble generating a tag: URI with plain XSLT so I looked around for an Atom ID HOWTO but then I found an interesting article which mentions that tag: isn’t a registered schema and argues that if your URIs don’t change anyway then there’s no need for a tag: URI. As a result my atom:id element contains the same URI as atom:link@atom:href (mmm, pseudo XPath) does.
