The site

I believe IE is being given application/rdf+xml when it visits my home page (or so I’ve been told) because it’s a moron. For that, I apologise. As far as I can tell only IE is stupid enough to fetch RDF instead of something more useful.

I’m not sure I can change the behaviour since I can only edit .htaccess.

I tried

AddType text/html .html; qs=0.9
AddType application/xhtml+xml .xhtml; qs=0.8
AddType application/rdf+xml .rdf; qs=0.7

(see also) but it doesn’t seem to work when using

HEAD / HTTP/1.1
Host: holizz.com
Connection: close

as it just gives me application/rdf+xml. Maybe IE would work with it. Maybe not.

Getting rid of Index.rdf isn’t really an option as I want RDF-capable agents to be fed RDF if they choose.

If IE is badly-behaved and I can’t make it behave well, I will not lose sleep. Just readers. But ah well.

Next up: a new style.

Yes, look at my lovely stylesheet. I was going to go for a sea theme, but the sky looked better than the sea in the photo I used. I found it via Creative Commons Search.

It’s surprisingly hard to find nice graphics to use. And no, I am not an 1337 GIMPerz so I can’t make my own.

If you’ll notice, the div#header element has been instructed overflow: auto to stop it from overflowing and making the rest of the page unreadable if it’s scrolled up. You may also notice that the font is 120% (well over the 100% the W3C advises) thus facilitating readability. For the same end there is no junk like a copyright statement and blogroll to clutter up the pages.

An finally: my email address.

As you may have noticed, my contact page includes my incomplete email address for spam concerns. I had a thought which involved embedding the agent’s IP address in the address (since I get email to all at this domain). Thus it would give mailto:127.0.0.1@this_domain (I’m still not brave enough), for example. And each person or harvester would get a different address. Humans and machines well-versed in English will be informed that they can just email holizz@… if they prefer.

I’m not sure about the idea though as I don’t know if it would be very useful in practice. It would allow me to give real URIs out (which is nice) yet it could just result in a lot of spam to a lot of different email addresses.
