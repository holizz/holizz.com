UTF

So I had this KSCP coursework. It was mostly multiple-choice, based on content from the course text with a couple of fun questions at the end. And by fun I mean more than five possible answers.

The first one asks: Briefly, describe a topic from the book which you found interesting.

My response to this was as follows; [the book being Computer Confluence (concise fifth edition) by George Beekman]

The book wrongly claims: The emerging international standard double-byte character set called Unicode… is designed to facilitate multilingual computing . My piece is on an encoding of Unicode which, unlike the one described in the book, exists (UTF-16 uses two bytes per character, except when it doesn’t).

UTF-8, 8-bit Unicode Transformation Format (the most commonly used Unicode encoding), uses a variable number of bytes per character depending on the code point being depicted. One byte is used for code points in the range of U+0000 to U+007F. With one-byte characters, UTF-8 is identical to US-ASCII. UTF-8 characters may have from one to four bytes depending on their position in Unicode. One advantage of using UTF-8 is that it can represent characters in a way that’s backwards compatible with US-ASCII using one byte per character but also represent any other character in Unicode. This is a large advantage in authoring texts which mostly use characters in Basic Latin (the section of Unicode which is equivalent to US-ASCII), such as English, French, Esperanto. However, in texts requiring more and higher code points (i.e. in Chinese or Japanese text) UTF-8 can produce larger files than other encodings such as UTF-16 or UTF-32.

I completely neglected how UTF-8 works, but the question was only worth three marks so neh.

I then go on to discover that one of the multiple choice questions is wrong (post submission, unfortunately):

The Unicode coding system can provide how many different representations?

  • 2
  • 4
  • 16
  • 256
  • about 65,000

I thought 65,000 was a little low but it’s the only one that makes the slightest bit of sense (it’s also the answer the book gives) so I go ahead and circle the fifth answer.

Later on in the day, inspired by the recent spate of UTF-related swoting (mostly on the Wikipedia article UTF-8) I was reading an article called UTF (on the subject of UTF, strangely) by Sun-folk Tim Bray in which a few particular sentences caught my attention: [emphasis mine]

Unicode currently defines just under 100,000 characters, but has space for 1,114,112 code points. They are organised into 17 planes of 216 (65,536) characters, numbered 0 through 16.

So there are 100,000 characters at the moment, with a possibility of 1,114,112 and 65,536 characters in each plane. So the actual answer is 1,114,112.

The moral of this story is that you should never trust what you read. Ever.
