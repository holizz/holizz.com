Accesskeys: how I hate them and how I disable them

Today I was directed to a nice page on the W3C’s site: List of valid DTDs you can use in your document. Since I recently switched to ratpoison, I have my display laid out with Firefox taking up the bottom half thus I removed the bookmarks toolbar thus I access the bookmarks menu with alt-b. I wanted to alt-b to the bookmarks menu so I could activate my del.icio.us bookmarklet. This is a problem when a Web page defines accesskeys because they override the ability to access menus from the keyboard. There’s no issue of me not being able to access links from the keyboard as I have Hit-a-Hint installed (i.e. I hold the space bar and type a combination of letters to click a link).

From a Web designer’s point of view I have decided that accesskeys are good for accessibility in some cases (i.e. users who don’t have Hit-a-Hint or similar installed and don’t mind having their keys overridden) but I will not implement them until the vast majority of browsers can turn them off easily (i.e. until Firefox adds an option in the Preferences dialog to disable them).

Here’s an interesting article on accesskeys and how they hurt accessibility: Leave my keys alone!

I’ve worked out how to disable accesskeys: go into about:config (i.e. type about:config into the address bar) and set ui.key.generalAccessKey to 0 (i.e. right-click it > Modify > set it to 0). That disables accesskeys in documents but still allows you to access the menus (you can disable the menu by doing the same thing to ui.key.menuAccessKey).

Ratpoison

Right now I am using Ratpoison. It’s a window manager. There are no pretty decorations, no wavy-cool menus popping up everywhere. But it’s special, and do you know why?

Because…

It is largely modelled after GNU Screen

I read that sentence and I knew I wanted it.

Screen is the best thing since, well… computers. I don’t know how I survived before I discovered screen. My first two commands upon logging into X are xterm -e screen and xterm -e screen -x.

I’m writing this entry in vim in screen in an xterm in ratpoison in X11. Sometimes it gets a little confusing navigating all my different things. Spending a long time in screen (like I do every day) sometimes makes me use C-A in Firefox to switch tabs, which doesn’t work (there should be a screen-alike extension for Firefox). It can get tedious when I’m sshed to another host using screen there inside my screen here, which have the same control character because I’m too lazy to change it there.

Don’t forget to check out the pictures of supported hardware (i.e. pictures of keyboards people use with ratpoison). I especially like the datahand and the modified Dvorak Happy Hacking keyboard.
