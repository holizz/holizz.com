User script to fix iPac

I finally found a useful purpose for JavaScript: fixing crappy sites.

As you may know, Bradford University has a library. What you may not know is that there is a web-based interface to that library based on some terrible software known only as iPac (no, I have no useful link, I’m not sure it even has a site). Apparently it’s been renamed and it does have a website: Horizon Information Portal.

So what I did tonight was write a Greasemonkey user script (although I’m sure it would be trivial to port it to Opera’s UserJS) that would fix iPac. I covered my two main gripes: firstly, a search field is always focused when the page loads making me tab twice before I can use Hit-a-Hint or navigate; secondly, the site uses javascript: links for no good reason (tip: there is no good reason). Using javascript: links is evil because it means I can’t do anything useful with the link like open it in a new tab.

Here it is: iPac Fixer.

Bug reports and patches appreciated (to userjs@holizz.com).

The user script basically sets the body’s onload attribute to an empty string to prevent the focus being set. Then it looks through all a elements and finds any with a href attribute starting with javascript: with single quotes, it then gets what’s between the first two single quotes and replaces the href with that.

MediaWiki bug-reporting

Today I filed two bug reports with MediaWiki, the popular wiki engine that powers great projects like the famous Wikipedia and the infamous BrafordCompski Wiki.

They also happen to be my first two bug reports ever! I am no longer a bug-report virgin!

They’re both related to the Atom syndication format and MIME types.

The first, Bug 2392: Atom items given text/plain MIME type, is an annoying bug that means when I read the Recentchanges Atom feed, I see the HTML source of the items rather than formatted data in HTML. Not a major idiocy, it just shows nobody actually uses the Atom feed (this would make sense as the RSS feed is listed first and thus would be chosen over Atom by most readers which autodiscover feeds).

The second, Bug 2393: Incorrect MIME type used to advertise Atom feed in Recentchanges, made me giggle when I first noticed it. Again, nothing serious but it could cause autodiscovery problems if the software looks for application/atom+xml specifically.
