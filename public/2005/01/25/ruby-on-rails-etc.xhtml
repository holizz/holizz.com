Ruby on Rails, etc.

The Ruby on Rails home page states:

Rails is a full-stack, open-source web framework in Ruby for writing real-world applications with joy and less code than most frameworks spend doing XML sit-ups

Full-stack means nothing to me. Framework could mean anything. Real-world applications? As opposed to fantasy applications?

From the description I know that Rails is something to do with Ruby and the Web. Really helpful.

So earlier today I spent a good while trying to install Rake, Gem, and Rails.

I eventually found this tutorial and this slightly more relevant one.

I followed the advice of Rails and gave it a database and stuff. I then created a thing called ‘blog’. It didn’t work, of course, when I visited the blog thing at /weblog/. It didn’t tell me anything helpful.

So I now have Ruby on Rails installed and I’m still in the dark as to what it actually is. I assume that it’s something like Zope that runs on magic.

By Space! I just found an amazing Web site: ABC News. Not a table used for layout in sight! It’s not amazing because it’s slightly accessible, it’s amazing because it’s a slightly accessible Web site of a big “they’ll think what I tell them to” corporation. It also tries (but fails) to be XHTML 1.0 Transitional. And if that wasn’t enough, it has CSS drop down menus! See!
